# copywriting - clear, specific, persuasive

The promise: copy that a skimming reader understands and believes in seconds - benefit-led, backed by proof, in the customer's language, not vague hype.

## When to use / when NOT
- **Use**: headlines, hero/landing copy, value props, feature descriptions, CTAs, emails, in-product microcopy.
- **Not**: positioning/strategy, or CRO layout/experiments (separate concerns; deeper GTM - positioning, CRO, SEO, pricing - ships in PALATOS Pro).

## Inputs
- Product, target reader, their problem, the top differentiators, and any proof (numbers, names, outcomes). Pull from `.palatos/company.md` if present.

## Method
1. **Nail the one message** - the single most valuable, true, specific thing to this reader. Everything supports it.
2. **Lead with benefit, back with feature** - what the reader gets → how it's delivered. Translate features into outcomes.
3. **Build the hierarchy** - headline (the promise) → subhead (how/for whom) → body (proof + specifics) → CTA (one clear next action).
4. **Use their words** - mirror the customer's phrasing of the problem; drop internal jargon and stack-speak.
5. **Add proof** - specifics, numbers, names, outcomes beat adjectives. Replace "powerful" with the fact that makes it powerful.
6. **Cut** - remove filler, hedges, and any sentence that doesn't advance the message. Short words, active voice.
7. **Readability gate** - scannable (short paragraphs, meaningful subheads), grade-8 reading level unless the audience is technical, one idea per sentence.
8. **Give variants** - 2–3 headline/CTA options to test.

## Output contract
```
COPY - <page/asset>
message: <the one thing>
headline: <primary> | variants: <2-3>
subhead: <one line>
body: <benefit-led, proof-backed>
cta: <primary action> | variants
notes: <tone, proof used, what to A/B>
```

## References (progressive disclosure)
- `references/playbook.md` - headline formulas, page-structure framework, CTA library, page-specific + voice/tone guidance.

## Quality bar
- A skimmer gets the value in <5 seconds.
- Every claim is specific and true; proof over adjectives.
- One message, one primary CTA.
- Passes the readability/scannability gate.

## Self-check rubric
- [ ] Is there one clear message and one primary CTA?
- [ ] Is it benefit-led with real proof (not adjectives)?
- [ ] Did I cut filler/jargon and keep it scannable?
- [ ] Did I give testable variants?

## Anti-patterns
- Feature lists with no benefit; adjective soup ("powerful, seamless, robust").
- Burying the value below the fold of the sentence.
- Multiple competing CTAs; internal jargon.
- Clever over clear.
