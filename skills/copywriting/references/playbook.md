# Copywriting playbook - full depth

Loaded by the `copywriting` skill when writing page copy. Captures the complete framework (formulas, page structure, CTA library, page-specific guidance) plus the PALATOS discipline layer: self-check gate, no-fabrication guardrail, and variant discipline.

## Context to gather first
Read `.palatos/company.md` and `.palatos/positioning.md` before asking. Then confirm only what's missing:
- **Page purpose** - type (homepage / landing / pricing / feature / about) + the ONE primary action.
- **Audience** - ideal customer, their problem, objections/hesitations, and the *words they use* for the problem.
- **Offer** - what's sold, the key transformation/outcome, differentiators, proof points.
- **Context** - traffic source (ad / organic / email) and what the visitor already knows on arrival.

## Writing-style rules
1. Simple over complex ("use" not "utilize").
2. Specific over vague (kill "streamline / optimize / innovative / seamless").
3. Active over passive.
4. Confident over qualified (cut "almost / very / really").
5. Show over tell (describe the outcome, drop adverbs).
6. Honest over sensational - **never fabricate stats or testimonials** (erodes trust + legal risk).

## Headline formulas
- `{Achieve outcome} without {pain point}`
- `The {category} for {audience}`
- `Never {unpleasant event} again`
- `{Question that names the main pain}`
- Add specificity: numbers, timeframes, concrete detail. Social proof variant: "Join 10,000+ teams who…".

## Page-structure framework
**Above the fold:** Headline (single most important message) -> Subheadline (expand + specificity, 1-2 sentences) -> Primary CTA (communicate what they get).

**Core sections (in a logical top-to-bottom argument):**
| Section | Purpose |
|---|---|
| Social proof | Credibility (logos, stats, testimonials) |
| Problem / pain | Show you understand their situation |
| Solution / benefits | 3-5 key benefits tied to outcomes |
| How it works | Reduce perceived complexity (3-4 steps) |
| Objection handling | FAQ, comparisons, guarantees |
| Final CTA | Recap value, repeat CTA, add risk reversal |

## CTA library
- **Weak (avoid):** Submit, Sign Up, Learn More, Click Here, Get Started.
- **Strong (use):** Start Free Trial, Get [Specific Thing], See [Product] in Action, Create Your First [Thing], Download the Guide.
- **Formula:** `[Action verb] + [what they get] + [qualifier if needed]`.

## Techniques
- **Be direct** - get to the value; don't bury it in qualifications.
- **Rhetorical questions** - "Tired of chasing approvals?" pull the reader into their own situation.
- **Analogies** - make abstract concrete.
- **Humor** - only if on-brand and never at clarity's expense.

## Page-specific guidance
- **Homepage** - serve multiple intents without going generic; lead with broadest value; clear paths for different visitors.
- **Landing** - single message, single CTA, match the ad/source, complete the argument on one page.
- **Pricing** - help them choose; answer "which is right for me?"; make the recommended plan obvious. (Pricing strategy + CRO layout ship in PALATOS Pro.)
- **Feature** - feature -> benefit -> outcome; use cases; clear path to try/buy.
- **About** - why you exist, mission -> customer benefit, still include a CTA.

## Voice & tone
Establish formality (casual / professional-friendly / enterprise) and brand personality (playful vs serious, bold vs understated, technical vs accessible). Headlines can be bolder; body clearer; CTAs action-oriented. Keep consistent with `.palatos/company.md` brand.

## Output (extends the SKILL output contract)
Page copy by section + **annotations** (why each choice, which principle) + **2-3 alternatives** for headlines/CTAs with rationale + **meta** (SEO title + description) when relevant. SEO and deeper page-strategy (CRO) ship in PALATOS Pro.
